Nomination Statement: I am self-nominating for a seat on the Paketo Buildpacks Steering Committee.

I am a maintainer in the Paketo community, with current responsibilities across the Java, App Monitoring, Content, and Utilities teams. My day-to-day focus has been around the Java ecosystem: keeping the Java buildpacks useful and current for real applications, improving the Spring Boot experience, helping new JVM releases land quickly and safely, and making the project easier to understand and contribute to.

I originally came to buildpacks as a practitioner looking for a better way to build container images for Java applications than maintaining Dockerfiles and custom CI scripts. Since becoming a maintainer, I have contributed code, reviews, releases, documentation, blog posts, discussions, and user-facing examples. Some of the work I am proud of includes supporting the ongoing evolution of the Java buildpacks, communicating latest Java versions support, documenting how contributors can manually test buildpack changes, explaining Java buildpack enhancements, and helping the community understand Spring Boot performance options such as CDS/AOT Cache, native image, virtual threads, and recent JVM improvements.

I also spend a lot of energy outside GitHub advocating for Paketo and Cloud Native Buildpacks in the Java, Spring, Cloud Foundry, and Kubernetes communities. I speak about Paketo at conferences and meetups, answer questions from users, and try to bring that feedback back into the project. As president of the Montréal Java User Group and Devoxx4Kids Québec, and through my involvement with CNCF Eastern Canada, I care deeply about making open source projects welcoming, understandable, and sustainable.

If elected, I would focus on three things:

 - keeping Paketo reliable for production users, especially around timely Java/JVM, Spring, and security-related updates;
 - introducing security related features across the Paketo ecosystem: CVEs monitoring, more base images choices, SLSA compliance, etc;
 - and strengthening the bridge between Paketo maintainers and the broader developer communities that depend on buildpacks every day.
Thank you for reviewing my application!
